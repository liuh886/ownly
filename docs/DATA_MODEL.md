# Data Model

Ownly uses your Obsidian Vault as the database. Every record is a plain `.md` file with YAML frontmatter.

## Object Types and Schema

Ownly manages entities using the following types:

1. **Physical Objects**
   - Current CLI/Type: `physical` (Note: `physical_asset` is legacy only and should not be generated by new tools).
   - Fields: `id`, `object_type`, `title`, `purchase_price`, `total_acquisition_cost`, `category`, `status` (`observing`, `purchased`, `using`, `idle`, `transferred`, `discarded`), `purchased_at`, `ended_at`.

2. **Recurring Costs (Subscriptions)**
   - Type: `recurring_cost`
   - Fields: `id`, `object_type`, `title`, `billing_amount`, `billing_cycle`, `billing_day`, `payment_account`, `status` (`active`, `paused`, `cancelled`).

3. **One-Time Experiences**
   - Type: `one_time_experience`
   - Fields: `id`, `object_type`, `title`, `planned_at`, `started_at`, `ended_at`, `reviewed_at`, `budget_total`, `actual_total`, `expense_items`, `status` (`planned`, `in_progress`, `completed`, `reviewed`).

4. **Reviews**
   - Type: `review`
   - Fields: `id`, `review_type`, `target_id`, `reviewed_at`, `summary`, `scores` (food, scenery, experience), `ranks`.

5. **Snapshots**
   - Type: `snapshot`
   - Fields: `id`, `type: snapshot`, `snapshot_at`, `asset_balances`, `liability_balances`, `total_assets`, `total_liabilities`, `net_worth`.

## How Agents Read/Write Safely

To safely interact with Ownly data, **agents MUST use the CLI** (`npm run wyqd --`).
Do not bypass the CLI to directly edit Markdown files via file manipulation tools, as it risks corrupting the strict YAML frontmatter schemas.

The CLI validates the schema before writing. See `docs/AGENT_CLI_GUIDE.md` for CLI usage.
